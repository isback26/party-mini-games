## 3. 369 규칙 상태 정리

### easy
- 기존 369 규칙
- 숫자 안에 3/6/9가 들어가면 박수
- 아니면 숫자 입력

### normal
- easy 규칙 포함
- 10, 20, 30 ... 처럼 10의 배수에서는 `만세`

### hard
- normal 규칙 포함
- 10의 배수는 `만세`
- 숫자 안에 3/6/9가 들어가면 그 개수만큼 박수
- 그 외에도 **3의 배수는 숫자 대신 박수 1회**

예:
- `3` → 박수 1회
- `33` → 박수 2회
- `369` → 박수 3회
- `10` → 만세
- hard에서 `12` → 박수 1회

## 4. 현재 369 입력/UI 상태 정리

### 입력 방식
- 숫자 입력은 자동 제출 구조
  - 1~9는 1자리 입력 즉시 제출
  - 10 이상은 필요한 자리 수가 차면 자동 제출
- 박수 입력은 연속 탭 기반
- 만세 입력은 전용 버튼 1회 입력
- 정답 타입/횟수를 UI가 직접 알려주지 않음

### 표시 방식
- 상단 우측은 `정답`이 아니라 **직전 플레이어의 실제 입력값** 표시
- `waiting_start` 중에는 상단 우측에 카운트다운 표시
- 카운트다운 종료 후 직전 입력 표시로 전환

### 버튼 배치
현재 369 입력 패널은 아래 순서로 정리되어 있다.
1. 숫자 입력창
2. 숫자패드
3. 박수 입력 / 만세 입력
4. 뒤로(백스페이스) / 전체삭제

이 배치는 스마트폰 실사용 기준으로 손가락 이동을 줄이기 위한 방향이다.

## 5. 오답/종료 흐름 정리
현재는 오답 입력 시 단순 종료가 아니라 아래처럼 동작한다.

- 실제 잘못 입력한 값이 서버 metadata에 남는다
- 상단 우측 표시도 마지막 실제 입력값 기준으로 유지된다
- 종료 메시지에서 정답과 오답을 함께 알려준다
  - 예: `정답은 "👏👏"인데 틀렸습니다.`
- 게임 종료 대화상자에 **다음 게임은 누구부터 시작인지** 표시된다

시간 초과 패배도 동일한 “다음 시작 플레이어” 계산 흐름에 반영된다.

## 6. 시작 플레이어 규칙 정리
- **첫 게임**은 방장이 시작 버튼을 누르고, 첫 턴도 기본 순서 시작
- **다음 게임부터**는 직전 게임에서 틀리거나 시간 초과로 패배한 사람이 첫 턴이 된다
- 현재 단계에서는 **시작 버튼 권한은 여전히 방장**에게 있다
- 즉, 바뀐 것은 “누가 시작 버튼을 누르느냐”가 아니라 “첫 턴이 누구냐”이다
