## 3. 이번 채팅 기준으로 확인된 369 규칙 상태

### easy
- 기존 369 규칙
- 숫자 안에 3/6/9가 들어가면 박수
- 아니면 숫자 입력

### normal
- easy 규칙 포함
- 10, 20, 30 ... 처럼 10의 배수에서는 `만세`

### hard
- normal 규칙 포함
- 10의 배수는 `만세`
- 숫자 안에 3/6/9가 들어가면 그 개수만큼 박수
- 그 외에도 **3의 배수는 숫자 대신 박수 1회**

예:
- `3` → 박수 1회
- `33` → 박수 2회
- `369` → 박수 3회
- `10` → 만세
- hard에서 `12` → 박수 1회

## 4. 이번 채팅 기준 369 입력/UI 상태 정리

이번 창 동안 아래 방향이 정리되었다.

### 현재까지 도달한 방향
- 입력 화면은 스마트폰 기준으로 **숫자패드 + 빠른 입력 버튼** 구조를 유지한다.
- `만세 입력` 버튼이 추가되는 방향으로 정리했다.
- 로비에는 **369 전용 난이도 선택 UI**를 붙이는 방향으로 정리했다.
- 서버는 난이도 값을 받아 실제 게임 시작 시 반영하는 구조를 사용한다.

### 이번 창에서 중요하게 확인한 UX 이슈
- `숫자 제출` 버튼은 스마트폰에서 불편하다.
- 박수 1/2/3회 선택을 별도 선택창으로 고르는 것도 실전에서는 불편할 수 있다.
- 실제 게임에서는 **즉시 입력형에 가까운 구조**가 더 자연스럽다.
- 상단 우측에는 정답 표시보다 **직전 플레이어가 실제로 입력한 값**을 보여주는 방향이 더 맞다.

## 8. 다음 채팅에서 확정된 우선 방향

다음 채팅에서는 아래를 **한 번에 다 하지 않고 작은 단계로** 진행한다.

### 1순위
상단 우측 정답 표시 제거 → **직전 플레이어 입력 표시**로 바꾸는 설계를 먼저 시작한다.

### 2순위

목표 방향:
- 1~9는 숫자 누르면 즉시 제출
- 10 이상은 2자리 누적 후 자동 제출
- 턴 시간 안에 2번째 자리를 못 누르면 서버 timeout으로 종료
- normal / hard에서 십단위 만세 규칙과 충돌하지 않게 설계

### 3순위
박수 입력을 연속 탭 기반으로 재설계

목표 방향:
- 박수 1회가 정답이면 버튼 1번으로 즉시 제출
- 박수 2회 필요 시 2번 입력이 완료될 때 제출
- 박수 3회도 동일
- 시간 초과는 서버가 판정

### 4순위
`999` 도달 시 강제 종료 룰 추가

목표 방향:
- `999`까지 성공하면 더 이상 진행하지 않음
- `참 잘 했어요!` 메시지와 함께 게임 종료

## 12. 2026-03-28 추가 업데이트 (입력 UX 테스트 후 반영할 변경사항)

이번 채팅에서는 실제 스마트폰 2대 APK 테스트를 기준으로 369 입력 UX를 한 단계 더 진행했다.

### 이번 채팅에서 실제로 진행한 것
- 상단 우측 표시를 `정답 표시`가 아니라 `직전 입력 표시`로 바꾸는 방향을 적용 시작
- `숫자 제출` 버튼 제거
- 숫자 자동 제출 구조 적용
  - 1~9는 즉시 제출
  - 10 이상은 필요한 자리 수가 모이면 자동 제출
- 박수 선택창 제거
- 박수 연속 탭 누적 입력 구조 적용 시작
- LAN IP 변경으로 인한 스마트폰 서버 연결 문제 원인 확인
  - 테스트 시점에 PC 서버 IP가 `192.168.45.170`에서 `192.168.45.110`으로 변경됨
  - APK 내부 접속 주소도 함께 갱신되어야 한다는 점 확인

### 이번 채팅 기준으로 확인된 결과
- 숫자 자동 제출은 정상 동작 확인
- 박수 연속 탭 누적 입력도 기본 동작은 확인
- 다만 실제 플레이 감각상 아래 UX 문제들이 새로 확인되었다

### 새로 확인된 UX 문제
1. 박수 입력 턴에서 버튼에 `0/2`처럼 필요한 횟수를 암시하는 표시는 게임을 너무 쉽게 만든다.
2. 박수 입력 턴에서 숫자 키패드를 막아버리면 안 된다.
   - 참가자가 일부러 잘못된 숫자를 누를 수도 있어야 한다.
   - 즉, UI가 정답 타입을 직접 알려주면 안 된다.
3. 오답 입력 시 지금처럼 바로 종료만 하지 말고,
   - 실제 잘못 입력된 값이 상단/결과에 표시되어야 하고
   - `정답은 "()"인데 틀렸습니다.` 형태로 알려준 뒤 종료되는 흐름이 더 맞다.
4. 첫 게임 시작은 방장이 하되,
   - 다음 게임부터는 직전 라운드에서 틀린 사람이 다시 시작할 수 있게 바꾸는 방향이 필요하다.

- 최종 정답/오답 판정은 여전히 서버가 담당

### 2순위
**오답 입력 결과 표시 개선**

핵심 방향:
- 플레이어가 실제로 잘못 입력한 숫자/박수/만세를 서버 metadata에 남긴다
- 게임 종료 시 `정답은 "..."인데 틀렸습니다.` 메시지를 제공한다
- 상단 우측 또는 결과 영역에 마지막 실제 입력값이 보이도록 정리한다

### 3순위
**다음 라운드 시작권 변경**

핵심 방향:
- 첫 게임은 기존대로 방장이 시작
- 이후 재시작부터는 직전 패배자가 시작할 수 있도록 서버 state 또는 room state에 시작 플레이어 정보를 유지하는 구조 검토
- 이 변경도 서버 authoritative 흐름 안에서 처리

### 4순위
**999 종료 룰 추가**
- `999`까지 성공 시 `참 잘 했어요!` 메시지와 함께 종료
