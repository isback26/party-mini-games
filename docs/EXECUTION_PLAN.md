# EXECUTION_PLAN.md (v0.1)

파티 미니게임 앱 개발 실행 계획서

이 문서는 프로젝트의 **Single Source of Truth(SSOT)** 역할을 하며 개발 진행 중 지속적으로 업데이트된다.

---

# 1. 프로젝트 개요

## 프로젝트 목표
친구 초대 기반으로 즐길 수 있는 **실시간 파티 게임 앱**을 개발한다.

주요 사용 상황

- 친구들 술자리
- 가족 모임
- 청소년 온라인 놀이

핵심 특징

- 친구 초대 기반 멀티플레이
- 짧고 빠른 게임
- 간단한 규칙
- 누구나 쉽게 참여 가능

---

# 2. 게임 로드맵

## MVP (1차 출시)

### 369 게임

기본 규칙

숫자를 순서대로 진행하며
3 / 6 / 9가 포함된 숫자에서는 박수를 친다.

포함 개수만큼 박수 입력

예

1
2
👏
4
5
👏
7
8
👏
10
11
12
👏

시작 구호

"삼육구, 삼육구"

---

## 2차 업데이트

### 눈치 게임

규칙

목표 숫자까지 진행하며
한 턴에 1~3 숫자를 진행 가능

마지막 숫자를 말한 사람이 패배

시작 구호

"시작!!"

---

### 번데기 게임

패턴 규칙

1회
뻔 데기
후렴

2회
뻔 데기 뻔 데기
후렴

3회
뻔 데기 뻔 데기 뻔 데기
후렴

이후 반복 횟수 증가

시작 노래

"게임은 시작됐다
확률은 2분의1
이~히히
이~히히"

서버는 현재 패턴 상태를 계산하여 정답을 판정한다.

# 3. 시스템 구조

├─ Nunchi Game
├─ Bbeondegi Game
└─ Baskin Robbins


공통 엔진 역할

- 플레이어 관리
- 턴 관리
- 타이머
- 입력 처리
- 서버 판정
- 결과 처리

게임 모듈 역할

- 게임 규칙
- 정답 계산
- 상태 전이

# 4. 게임 시작 연출 시스템

각 게임은 시작 시 **구호 또는 노래 연출**을 가진다.

구성

startCueText
startAudio
startDelay

예

369
"삼육구 삼육구"

눈치 게임
"시작!!"

번데기
"게임은 시작됐다 확률은 2분의1 이~히히 이~히히"

서버는 게임 시작 이벤트를 보내고
클라이언트는 연출 이후 첫 턴을 시작한다.
